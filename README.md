# 接口自动化巡检Agent - 项目设计文档

## 1. 项目概述

### 1.1 项目背景

在后端日常运维、版本迭代、回归测试场景中，传统接口巡检存在大量痛点：人工逐个调用接口、核对返回结果、记录异常、汇总报告，效率极低、漏测率高、重复性工作多，且无法做到全天候定时巡检、智能分析异常根因。

常规自动化脚本仅能实现固定请求、固定断言，**不具备智能分析、自适应重试、异常推理、报告总结能力**。

基于以上痛点，本项目实现**Java 原生接口自动化巡检 AI Agent**，不依赖重型 AI 框架，纯后端工程化实现，完成「接口批量调用、智能断言、异常捕获、重试容错、AI 根因分析、自动生成巡检报告、异常告警」的完整闭环。

### 1.2 项目定位

后端专属**运维自动化 + AI 智能巡检**实战项目。

区别于普通自动化测试项目，本项目核心是 **Agent 智能决策**：

- 不是固定脚本跑用例，而是 AI 动态判断接口状态、异常类型、失败原因
- 具备任务调度、失败重试、动态策略、智能总结能力
- 纯 Java 工程化实现，贴合后端日常开发、运维、迭代场景

### 1.3 核心目标

- 实现接口全自动批量巡检，替代人工回归测试、日常巡检
- 基于 AI 自动分析接口报错、超时、参数异常、业务逻辑异常
- 支持串行/并行任务调度、失败重试、超时熔断、容错兜底
- 自动生成结构化巡检报告、异常汇总、修复建议
- 支持定时巡检、手动触发巡检、单接口重试、批量复测

## 2. 需求分析

### 2.1 功能需求

#### 2.1.1 接口任务管理

- 支持录入、编辑、启用/禁用接口巡检用例
- 支持配置接口地址、请求方式、请求头、请求参数、超时时间、基础断言规则
- 支持接口分组：用户模块、订单模块、支付模块、系统模块等

#### 2.1.2 自动化巡检执行能力

- 手动触发单次巡检、定时周期巡检（Cron 表达式配置）
- 支持串行有序执行、并行快速执行两种模式
- 统一 HTTP 调用、超时控制、状态码校验、响应结果校验
- 失败自动重试机制，可配置重试次数与重试间隔

#### 2.1.3 AI Agent 智能分析能力

- Agent 自动识别异常类型：网络超时、4xx参数错误、5xx服务异常、业务返回码异常、响应结构异常
- AI 基于报错堆栈、返回信息、请求参数，推理异常根因
- 自动给出修复建议、排查方向
- 批量巡检完成后，AI 自动生成整体巡检总结、风险点汇总

#### 2.1.4 报告与告警能力

- 生成结构化巡检报告：成功数、失败数、异常率、耗时统计
- 记录每次巡检的请求日志、响应日志、错误日志
- 异常接口支持消息告警（可扩展钉钉/企业微信/邮件）

#### 2.1.5 会话与任务记忆

- 记录历史巡检任务，可回溯任意一次巡检详情
- 支持多轮对话问答：查询最近巡检异常、查询某类接口稳定性

### 2.2 非功能需求

- **稳定性**：支持大量接口批量巡检，超时熔断、线程隔离，避免任务雪崩
- **可用性**：大模型调用失败、网络异常有兜底，不中断整体巡检任务
- **可扩展性**：新增巡检策略、新的异常类型、新的告警渠道可插拔扩展
- **高性能**：线程池管理巡检任务，并行执行提升巡检效率

### 3.1 架构思想

采用 **Java 后端分层工程化 + Agent 经典闭环架构**：思考（LLM 决策）→ 行动（接口巡检）→ 观察（结果收集）→ 总结（AI 分析报告）。
全程后端主控，AI 只负责智能分析与总结，所有任务调度、权限、重试、熔断、日志全部由 Java 工程掌控，企业级安全可控。

### 3.2 四层整体架构

#### 1）接入层（Controller）

提供手动巡检触发、任务查询、报告查询、用例管理接口，接收前端请求，参数校验，调度 Agent 核心服务。

#### 2）Agent 调度核心层（核心）

任务分发、执行策略控制、AI 决策调度、结果聚合、异常闭环。是整个系统的中枢，负责串联「任务执行+AI分析+报告生成」全流程。

#### 3）工具能力层

封装所有可被 AI 调用的后端工具能力：

- HTTP 接口调用工具
- 接口结果断言校验工具
- 失败重试工具
- 异常分类统计工具
- 报告生成工具

#### 4）基础服务层

LLM 客户端、线程池任务调度、定时任务、数据库持久化、日志记录、异常处理、告警工具。

### 3.3 核心业务流程（Agent 完整闭环）

1. **任务触发**：手动触发 / 定时任务触发巡检
2. **任务加载**：从数据库读取所有启用的接口巡检用例
3. **策略判断**：Agent 根据配置选择串行/并行执行策略
4. **工具执行**：批量调用接口，执行超时控制、状态校验、业务断言
5. **失败重试**：对临时失败接口自动重试，过滤抖动性异常
6. **结果收集**：收集所有接口的成功、失败、超时、异常信息
7. **AI 智能分析**：Agent 将异常日志送入大模型，识别异常类型、推理根因、给出修复建议
8. **报告生成**：AI 自动汇总本次巡检整体情况，生成巡检总结与风险评估
9. **持久化+告警**：保存巡检报告与日志，异常接口触发消息告警

## 4. 核心模块详细设计

### 4.1 LLM 智能推理模块

统一封装大模型调用客户端，通过结构化 Prompt 约束模型输出：异常分类、根因分析、修复建议、总结报告。

核心能力：

- 异常智能分类：网络异常、参数错误、服务500、业务逻辑错误、权限错误
- 报错堆栈智能解读
- 批量数据自动汇总分析

### 4.2 接口巡检工具模块

系统核心执行工具，完全可插拔设计：

- **HttpInvokeTool**：统一 HTTP 请求封装，支持 GET/POST/PUT/DELETE，支持请求头、Body、参数、超时配置
- **AssertTool**：状态码断言、返回码断言、关键字包含断言、空值断言
- **RetryTool**：可配置重试次数、重试间隔，过滤瞬时网络波动

### 4.3 Agent 任务调度模块

核心调度中枢，控制整个 Agent 生命周期：

- 任务线程池隔离，避免任务阻塞
- 串行/并行动态策略切换
- 单任务失败不影响整体批次任务
- 超时熔断，防止单个接口卡死整个巡检流程

### 4.4 异常智能分析模块

区别于传统自动化脚本，本模块是 AI Agent 核心亮点：

将「请求参数+响应内容+报错信息+状态码」送入大模型，由 AI 自动判断：

- 是偶发网络问题还是代码 bug
- 是参数缺失、参数格式错误还是后端逻辑异常
- 是服务宕机、超时、数据库异常还是第三方接口问题

### 4.5 报告与告警模块

- 生成标准化巡检报告：任务耗时、总用例数、成功数、失败数、异常率
- 异常接口单独汇总，附带 AI 分析结论与修复建议
- 支持异常触发实时消息告警

### 4.6 定时任务模块

基于 Spring Schedule 实现 Cron 定时巡检，支持每日凌晨巡检、每小时巡检、版本迭代后自动巡检。

## 5. 数据库设计

### 5.1 接口巡检用例表（api_inspect_case）

|字段|类型|说明|
|---|---|---|
|id|bigint|主键|
|case_name|varchar|用例名称|
|api_url|varchar|接口地址|
|method|varchar|请求方式 GET/POST|
|request_header|text|请求头JSON|
|request_body|text|请求体JSON|
|timeout|int|超时时间(ms)|
|retry_count|int|失败重试次数|
|assert_rule|text|基础断言规则|
|group_name|varchar|接口分组|
|status|tinyint|0-禁用 1-启用|
|create_time|datetime|创建时间|
### 5.2 巡检任务记录表（api_inspect_task）

|字段|类型|说明|
|---|---|---|
|id|bigint|主键|
|task_no|varchar|任务编号|
|total_count|int|总用例数|
|success_count|int|成功数|
|fail_count|int|失败数|
|task_duration|int|任务耗时(ms)|
|ai_summary|longtext|AI巡检总结报告|
|task_status|tinyint|任务状态|

### 5.3 接口巡检日志表（api_inspect_log）

记录每一次接口调用的详细请求、响应、异常信息，用于 AI 分析溯源。

## 6. 核心接口设计

- **手动触发巡检**：POST /agent/inspect/start
- **停止巡检任务**：POST /agent/inspect/stop
- **查询任务列表**：GET /agent/inspect/task/list
- **查询AI巡检报告详情**：GET /agent/inspect/report/{taskId}
- **用例新增/编辑/删除**：POST /agent/inspect/case/save
- **AI 问答复盘**：POST /agent/inspect/chat

## 7. 技术栈选型

- **核心框架**：SpringBoot 3.5.x
- **数据库**：MySQL 8.0
- **持久层**：MyBatis-Plus
- **HTTP 调用**：OkHttp / Hutool Http
- **定时任务**：Spring Schedule
- **线程池**：自定义ThreadPoolExecutor
- **JSON 处理**：FastJSON2
- **AI 能力**：原生 HTTP 调用大模型（通义/星火/DeepSeek/Qwen 通用适配）

## 8. 项目核心亮点（面试/简历重点）

- **纯原生 Agent 实现**：不依赖 LangChain4j、SpringAI，手写 Agent 调度闭环，吃透「思考-行动-观察-总结」核心原理
- **后端主控、AI 辅助**：所有任务调度、重试、熔断、权限、日志由 Java 掌控，AI 仅做智能分析，企业级安全可控
- **智能化远超传统自动化测试**：普通脚本只会固定断言，本项目 AI 可自动分析报错根因、区分偶发/必现问题、给出修复方案
- **工程化成熟度高**：线程池隔离、超时熔断、失败重试、定时调度、日志溯源、报告汇总、异常告警
- **业务价值极强**：可直接落地企业日常运维、版本回归测试，真实降本增效

## 9. 项目扩展方向

- 接入 Redis 实现任务分布式调度、任务状态持久化
- 接入 RAG 知识库，学习历史 bug 记录，提升异常分析准确率
- 接入钉钉/企业微信告警，异常实时推送负责人
- 新增接口性能分析：响应时间统计、慢接口预警
- 重构为 SpringAI 版本，适配框架化开发模式
- 增加多环境管理：测试环境、预发环境、生产环境巡检隔离

## 10. 部署运行流程

1. 执行 SQL 脚本创建三张业务表
2. 配置数据库、大模型 API 密钥、线程池参数、重试策略
3. 启动 SpringBoot 项目
4. 录入接口巡检用例，开启定时任务
5. 手动触发巡检，查看 AI 自动生成的巡检报告与异常分析

## 11. 项目总结

本项目是**Java 后端最优质的 AI Agent 运维自动化实战项目**，区别于普通 CRUD 项目和纯 AI 脚本项目，既具备扎实的 Java 工程化能力（线程池、任务调度、熔断重试、数据库设计），又具备当前热门的 AI Agent 智能决策能力。项目场景真实、业务价值清晰、技术亮点突出，非常适合作为简历主打项目、面试实战讲解项目，同时可直接改造为企业内部自动化运维工具。
